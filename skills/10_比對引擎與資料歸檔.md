# Skill 10：持股比對引擎與資料歸檔

## 技術分類
資料處理邏輯（Comparator + State Management）

## 概述
每日抓取完成後，需要比對「今日持股」與「前一日持股」，計算差額，並將結果寫入多個輸出路徑。

---

## 一、持股比對引擎（comparator.js）

### 輸入
```
currentHoldings:  [{ stockCode, stockName, shares, weight }, ...]
previousHoldings: [{ stockCode, stockName, shares, weight }, ...]
```

### 輸出
```
[{
  stockCode, stockName, shares, weight,
  diffShares,         // 股數差（正=增持，負=減持）
  diffSharesPercent,  // 股數變化百分比
  diffWeight,         // 權重差（保留兩位小數）
  isNew               // 是否為新進榜
}, ...]
```

### 邏輯
1. **首次執行（無前日資料）**：全部標記為 `isNew: true`，差額等於當日數值。
2. **正常比對**：
   - 建立 `prevMap = Map<stockCode, holding>`
   - 遍歷 currentHoldings，逐筆查詢 prevMap
   - 找到：計算 `diffShares` 和 `diffWeight`
   - 找不到：標記為 `isNew: true`
3. **精度控制**：`diffWeight` 使用 `toFixed(2)` 避免浮點誤差。

## 二、資料歸檔邏輯（index.js）

### 輸出檔案清單
| 檔案                              | 格式     | 用途                 |
|----------------------------------|---------|---------------------|
| `state.json`                     | 原始持股  | 下次比對的 baseline   |
| `data/{dateStr}.json`            | 含差額   | 歷史資料備份          |
| `dashboard/public/data/latest.json`    | 含差額   | Dashboard 預設載入   |
| `dashboard/public/data/{dateStr}.json` | 含差額   | Dashboard 歷史查詢   |
| `dashboard/public/data/index.json`     | 日期陣列  | 前端日期選單         |

### 日期計算
```javascript
const twTime = new Date(now.getTime() + 8 * 60 * 60 * 1000);
if (twTime.getUTCHours() < 15) twTime.setUTCDate(twTime.getUTCDate() - 1);
while (twTime.getUTCDay() === 0 || twTime.getUTCDay() === 6) {
  twTime.setUTCDate(twTime.getUTCDate() - 1);
}
```
台灣時間 15:00 前：取前一交易日。跳過週末。

### index.json 防護機制
只有在至少一個 ETF 成功抓取時，才將日期加入 `index.json`：
```javascript
const hasAnySuccess = Object.values(dashboardState)
  .some(h => Array.isArray(h) && h.length > 0);
if (hasAnySuccess) { /* 登錄日期 */ }
```
防止全失敗時產生空殼日期，導致前端顯示空白頁。

### state.json vs dashboardState
| 欄位      | state.json | dashboardState |
|----------|-----------|----------------|
| 原始持股  | ✅         | ✅              |
| diffShares| ❌        | ✅              |
| diffWeight| ❌        | ✅              |
| isNew    | ❌         | ✅              |

`state.json` 只存原始資料（供下次比對），`dashboardState` 包含差額（供前端顯示）。

## 三、Puppeteer 共用瀏覽器模式

### 問題
每個 ETF 都啟動一個新的瀏覽器太慢且吃資源。

### 解法
```javascript
let sharedBrowser = null;
let sharedPage = null;

async function getSharedPage() {
  if (!sharedBrowser) {
    sharedBrowser = await puppeteer.launch({ headless: 'new', args: [...] });
    sharedPage = await sharedBrowser.newPage();
    // 設定 UA, viewport, webdriver 隱藏
  }
  return sharedPage;
}

export async function closeBrowser() {
  if (sharedBrowser) { await sharedBrowser.close(); }
}
```
所有需要瀏覽器的策略共用同一個 `page`，全部抓完後統一 `closeBrowser()`。

### 反偵測措施
```javascript
await page.evaluateOnNewDocument(() => {
  Object.defineProperty(navigator, 'webdriver', { get: () => undefined });
});
```
搭配 `puppeteer-extra-plugin-stealth` 隱藏 headless 特徵。
