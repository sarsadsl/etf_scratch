# Skill 07：分頁與路由系統（HashRouter + 分類篩選）

## 技術分類
React Router + 條件式渲染

## 為何選擇 HashRouter
GitHub Pages 不支援 server-side redirect（如 `historyApiFallback`）。若使用 `BrowserRouter`，使用者直接存取 `/industry/bmc` 時 GitHub Pages 會回 404。`HashRouter` 將路由放在 `#` 之後（如 `/#/industry/bmc`），完全在前端處理，不需要 server 配合。

## URL 結構
| 頁面             | URL                             |
|-----------------|--------------------------------|
| ETF 追蹤（預設） | `/#/`                           |
| 產業分析列表     | `/#/industry`                   |
| BMC 文章        | `/#/industry/bmc`               |
| 未來文章（範例） | `/#/industry/{articleId}`        |

## 路由設定（main.jsx）
```jsx
import { HashRouter } from 'react-router-dom';

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <HashRouter>
      <App />
    </HashRouter>
  </StrictMode>
);
```

## 路由定義（App.jsx）
```jsx
import { Routes, Route, useNavigate, useLocation } from 'react-router-dom';

function App() {
  const navigate = useNavigate();
  const location = useLocation();
  const isIndustry = location.pathname.startsWith('/industry');

  return (
    <div className="app-container">
      {/* 頂層 Nav */}
      <nav>
        <button onClick={() => navigate('/')}>ETF 追蹤</button>
        <button onClick={() => navigate('/industry')}>產業分析</button>
      </nav>

      <Routes>
        <Route path="/industry" element={<IndustryIndex />} />
        <Route path="/industry/bmc" element={<BmcPage />} />
        <Route path="/*" element={<EtfDashboard />} />
      </Routes>
    </div>
  );
}
```

## 頂層導航 Tab 設計
```javascript
const NAV_TABS = [

```
- 使用 `useLocation().pathname.startsWith('/industry')` 判斷 active 狀態
- 點擊時用 `navigate()` 跳轉，不使用 `<Link>`（因為需要自定義 button 樣式）

### 文章定義
```javascript
const ARTICLES = [
  {
    id: 'bmc',              // 對應 Route path
    category: 'semiconductor', // 對應 CATEGORIES.id
    title: '...',
    subtitle: '...',
    desc: '...',
    tags: ['信驊', 'BMC'],
    accentColor: '#06B6D4',
    date: '2026-04-28',
  },

```

### 篩選邏輯
```javascript
const filtered = ARTICLES.filter(a => {
  const matchCat = activeCategory === 'all' || a.category === activeCategory;
  const matchSearch = !query || a.title.includes(query) || a.tags.some(t => t.includes(query));
  return matchCat && matchSearch;
});
```
同時支援分類 Tab 篩選 + 關鍵字搜尋。

## 新增文章的步驟
1. 建立 `dashboard/src/pages/新文章.jsx` 元件
2. 在 `IndustryIndex.jsx` 的 `ARTICLES` 陣列新增一筆
3. 在 `App.jsx` 的 `<Routes>` 新增 `<Route path="/industry/{id}" element={<新文章 />} />`

## 已知注意事項
1. HashRouter 的 URL 帶有 `#`，SEO 不友好，但 GitHub Pages 環境下這是最佳解。
2. 未來若遷移至自有 server（如 Vercel），可改用 `BrowserRouter` + `_redirects` 檔案。
3. ETF 追蹤路由使用 `/*` wildcard，必須放在 Routes 的最後一個。
